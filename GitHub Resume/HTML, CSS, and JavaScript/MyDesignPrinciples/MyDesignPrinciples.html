Complexity Considerations

Complexity Principles
  • Tesler's Law (The Law of Conservation of Complexity): "For any system there is a certain amount of complexity which cannot be reduced!"
  • Allow users to learn the system by doing
  • Help users adopt more efficient methods
  • Provide fluid pathways
  • Reduce clutter without losing capability
  • Make important information visually salient
  • Don’t allow simplicity to turn into abstraction
  • Ensure that the meaning of an icon is obvious
  • Simplifying the user’s interaction requires more complexity behind the scenes
  • Complexity is incremental

Symptoms of Complexity
  • Simple changes require modifications in multiple places
  • A developer must have extensive knowledge of the code to complete a task
  • It is unknown what code must be modified for a specific task

Causes of Complexity:
  • Dependencies
  • Obscurity
  • Redundancy

Culteral Considerations

Remember the effects of...

  1. Individualism vs Communitarianism
  2. Universalism vs Particularism
  3. Specific vs Diffuse
  4. Neutral vs Emotional
  5. Achievement vs Ascription
  6. Sequential Time vs Synchronous Time
  7. Internal Direction vs Outer Direction
Cross-Cultural Design
  • Determine unique terms
  • Don’t rely on machine translation
  • Identify the market’s primary devices and connectivity

Human-Centered Design Principles

  • Provide discoverability
  • Provide Feedback
  • Utilize a conceptual model
  • Include affordances, signifiers, constraints
  • Ensure you have good mappings (the relationship between controls and their actions)

Conceptional Considerations

  • Design the conceptional model purposely
  • Be wary of a poor or erroneous model
  • Make sure the mental model and the conceptional model match
  • Ensure that metaphors have affordances and constraints

Four Basic Activities of Interaction Design

  • Discovering Requirements
  • Designing Alternatives
  • Prototyping
  • Evaluating

Five Key Issues to Focus on When Gathering Data

  • Setting Goals
  • Identifying Participants (and how many)
  • Relationship with Participants
  • Triangulation of Data/Investigator/Theories/Methodologies
  • Pilot Studies

Cognitive Design Principles

Be ready to design for...

  • Attention
  • Perception
  • Memory
  • Learning
  • Reading, speaking, and listening
  • Problem-solving, planning, reasoning, and decision-making

Emotional/Accessibility Principles

  • Know that Western cultures express and recognize emotions that have a higher level of arousal (fear, anger, happiness, etc)
  • Know that Eastern cultures express and recognize emotions with a lower level (boredom, contentment, sleepiness, etc).
  • Vision
    • Pay attention to font usage
    • Note text size and amount of text
    • Ensure that color is not the only highlight
    • Label the color (if color is important)
    • Design for multiple interfaces and devices
  • Know That Haptic Feedback is...
    • Passive
    • Active
  • Accessibility may address...
    • Temporary Limitation
    • Situational Limitation
    • Effects of Aging
    • Disabilities

Error Principles

  • Be aware that people will always make mistakes
  • Assume that something will go wrong
  • Remember the Five Why’s: Keep asking “Why?” until you find the root cause
  • Error Messages:
    • Tell what the user did
    • Explain the problem
    • Instruct how to correct it
    • Use plain language and active voice
    • Show example, if appropriate
